# CakeMarket - Маркетплейс для кондитеров

Платформа для самозанятых кондитеров, позволяющая продавать свои изделия без необходимости создания и раскрутки собственного сайта.

## 📋 Описание проекта

Кондитеры регистрируются на платформе, проходят модерацию у оператора (администратора) и могут выкладывать свои товары (торты, пирожные, десерты). Покупатели видят ближайших к ним кондитеров и могут делать заказы.

### 🔑 Ключевые особенности

1. **Геолокация** - покупатели видят сначала ближайших кондитеров
2. **Справедливое распределение заказов** - уникальная методика распределения между кондитерами
3. **Модерация кондитеров** - оператор подтверждает регистрацию кондитеров
4. **Система рейтингов** - отзывы и оценки качества работы
5. **Управление заказами** - полный цикл от создания до завершения

## 🎯 Методика справедливого распределения заказов

При создании заказа система автоматически выбирает лучшего кондитера на основе:

1. **Расстояние** (30% веса) - чем ближе, тем лучше
2. **Текущая загрузка** (40% веса) - учитываются активные заказы за последние 7 дней
3. **Рейтинг** (30% веса) - средняя оценка от покупателей

Это гарантирует:
- Все кондитеры получают заказы со временем
- Качество вознаграждается (лучший рейтинг = больше заказов)
- Эффективность доставки (близкие кондитеры в приоритете)
- Никто не перегружается заказами

## 🛠 Технологический стек

### Backend
- **FastAPI** - современный Python фреймворк
- **SQLAlchemy** - ORM для работы с базой данных
- **SQLite** - база данных (легко заменяется на PostgreSQL)
- **Pydantic** - валидация данных
- **Passlib** - хеширование паролей
- **python-jose** - JWT токены
- **Geopy** - расчёт расстояний между координатами

### Frontend
- **React** - UI библиотека
- **React Router** - навигация
- **Axios** - HTTP запросы
- **CSS3** - стилизация

## 🚀 Быстрый старт

### Backend

```bash
cd cake-marketplace/backend
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

API документация будет доступна по адресу: http://localhost:8000/docs

### Frontend

```bash
cd cake-marketplace/frontend
npm install
npm run dev
```

Приложение будет доступно по адресу: http://localhost:5173

## 👥 Роли пользователей

### Покупатель (Customer)
- Просмотр товаров с учётом геолокации
- Создание заказов
- Оценка выполненных заказов

### Кондитер (Confectioner)
- Регистрация с модерацией
- Добавление/редактирование товаров
- Управление заказами (статусы)
- Просмотр статистики и рейтинга

### Администратор (Admin)
- Одобрение регистрации кондитеров
- Просмотр всех заявок

## 📝 API Endpoints

### Аутентификация
- `POST /api/v1/register` - регистрация
- `POST /api/v1/login` - вход

### Товары
- `GET /api/v1/products` - список товаров (с опцией геолокации)
- `POST /api/v1/products` - создать товар (кондитер)

### Заказы
- `POST /api/v1/orders` - создать заказ
- `GET /api/v1/orders` - мои заказы
- `PUT /api/v1/orders/{id}/status` - обновить статус (кондитер)

### Админка
- `GET /api/v1/admin/confectioners` - заявки кондитеров
- `PUT /api/v1/admin/confectioners/{id}/approve` - одобрить кондитера

### Отзывы
- `POST /api/v1/orders/{id}/review` - оставить отзыв

## 💡 Как это работает

1. **Кондитер регистрируется** → ждёт подтверждения оператора
2. **Оператор одобряет** → кондитер может добавлять товары
3. **Покупатель ищет товары** → видит ближайшие предложения
4. **Покупатель создаёт заказ** → система автоматически выбирает кондитера по алгоритму
5. **Кондитер выполняет заказ** → меняет статусы
6. **Покупатель получает заказ** → оставляет отзыв
